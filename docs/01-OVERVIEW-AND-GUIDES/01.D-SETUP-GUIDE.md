# Setup and Configuration Guide

This guide provides a detailed explanation of all the environment variables used in the project. These variables are configured in a `.env` file in the project root. To get started, copy the template: `cp env.example .env`.

---

### Core Server & Environment Settings
| Variable       | Description                                                                                             | Default in `env.example` |
| -------------- | ------------------------------------------------------------------------------------------------------- | ------------------------ |
| `NODE_ENV`     | Sets the application environment. `demo` uses a mock KMS, `development` uses full crypto.               | `demo`                   |
| `PORT`         | The network port the server will listen on.                                                             | `3000`                   |
| `API_HOSTNAME` | The public-facing hostname of the server.                                                               | `localhost`              |

---

### Host Organization Bootstrap Details
These variables are critical for the initial setup. They define the identity of the "Host" organization, which is the root tenant of the gateway.

| Variable                  | Description                                                              | Example Value                  |
| ------------------------- | ------------------------------------------------------------------------ | ------------------------------ |
| `ORG_HOST_LEGAL_NAME`     | The official legal name of the host organization.                        | `Gateway Host Services`        |
| `ORG_HOST_JURISDICTION`   | The two-letter country code where the host is registered.                | `ES`                           |
| `ORG_HOST_ID_TYPE`        | The type of official identifier (e.g., `TAX`).                           | `vat`                          |
| `ORG_HOST_ID_VALUE`       | The value of the official identifier.                                    | `A0011223344`                  |
| `ORG_HOST_ADMIN_EMAIL`    | The email address of the host's primary administrator.                   | `admin@host.com`               |
| `ORG_HOST_ADMIN_UID`      | A stable, unique identifier for the administrator.                       | `host-admin-001`               |
| `ORG_HOST_ADMIN_ROLE`     | The administrator's role, using a standard like ISCO-08.                 | `ISCO-08:1111`                 |
| `ORG_HOST_TERMS_URL`      | A URL to the terms of service for the host.                              | `http://github.com/...`        |

### Security & Backend Configuration
| Variable         | Description                                                                                             | Default in `env.example`    |
| ---------------- | ------------------------------------------------------------------------------------------------------- | --------------------------- |
| `KEK_SECRET`     | **CRITICAL:** A Base64-encoded, 32-byte secret for the Key Encryption Key. **DO NOT use the example in production.** | `your-super-secret...`      |
| `QUEUE_PROVIDER` | The provider for the job queue. `mem` for in-memory (local dev).                                        | `mem`                       |
| `DB_PROVIDER`    | The storage provider. `mem` for in-memory, `firestore` for Google Cloud Firestore.                      | `firestore`                 |
| `STORAGE_PROVIDER` | The provider for file/blob storage. `gcs` for Google Cloud Storage.                                     | `gcs`                       |

---

### GCP Project & Credentials
These are required if you are using `firestore` or `gcs` as providers.

| Variable                        | Description                                                                                             | Example Value                     |
| ------------------------------- | ------------------------------------------------------------------------------------------------------- | --------------------------------- |
| `FIRESTORE_PROJECT_ID`          | The GCP project ID where your Firestore and GCS bucket are located.                                     | `your-gcp-project-id`             |
| `GCS_BUCKET_NAME`               | The globally unique name of your Google Cloud Storage bucket.                                           | `your-globally-unique-bucket-name`|
| `GOOGLE_APPLICATION_CREDENTIALS`| Path to your GCP service account JSON key file for local development.                                   | `./gcp-service-account.json`      |
| `FIREBASE_API_KEY`              | The Web API Key from your Firebase project settings, used for authentication tests.                     | `firebase-web-api`                |

---

### GCP Deployment Configuration
These variables are used by the `cloud_deploy.sh` script.

| Variable                   | Description                                                                             | Default in `env.example` |
| -------------------------- | --------------------------------------------------------------------------------------- | ------------------------ |
| `DEPLOY_REGION`            | The GCP region for the deployment (e.g., `europe-southwest1` for Madrid).               | `europe-southwest1`      |
| `DEPLOY_SERVICE_NAME`      | The public name for the service on Cloud Run.                                           | `gwtemplate`             |
| `ARTIFACT_REGISTRY_NAME`   | The name for the repository in Google Artifact Registry.                                | `gwtemplate-images`        |

---

### Build-Time Secrets
| Variable    | Description                                                                         | Example Value      |
| ----------- | ----------------------------------------------------------------------------------- | ------------------ |
| `NPM_TOKEN` | (Optional) A token for accessing private NPM packages during the `docker build`.    | `your-npm-token`   |
