# Deployment Guide: Google Cloud Run

This guide provides instructions for deploying the application to Google Cloud Run using the provided automation script.

## Overview

This project includes an automated script (`cloud_deploy.sh` in the root directory) to build and deploy the application as a containerized service on Google Cloud Run. This is ideal for creating staging, testing, or production environments.

The process is designed to be simple and repeatable, relying on a local `.env` file for configuration. For a complete guide on setting up your `.env` file, see the **[Setup Guide](01.D-SETUP-GUIDE.md)**.

## Prerequisites

1.  **Google Cloud SDK (`gcloud`)**: Ensure you have the [Google Cloud SDK](https://cloud.google.com/sdk/docs/install) installed and authenticated.
2.  **Docker**: Ensure Docker is installed and the daemon is running.

## Execution Steps

1.  **Configure `.env`**: Ensure your `.env` file contains the correct values for the target environment, especially the variables under `GCP Deployment Configuration`.

2.  **Make the script executable** (only needs to be done once):
    ```bash
    chmod +x cloud_deploy.sh
    ```

3.  **Run the deployment script** from the project root:
    ```bash
    ./cloud_deploy.sh
    ```

The script will automate all steps: enabling APIs, creating the Artifact Registry repository, building/pushing the image, and deploying to Cloud Run. Upon completion, it will output the public URL of the service.
